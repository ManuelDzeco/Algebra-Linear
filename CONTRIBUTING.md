# Guia de Contribuição

Obrigado por seu interesse em contribuir para o projeto **Álgebra Linear - Uma Abordagem Visual**! 

## 🤝 Como Contribuir

### Reportar Bugs

Se você encontrou um bug, por favor abra uma **Issue** com:
- Uma descrição clara do problema
- Passos para reproduzir
- Comportamento esperado vs. observado
- Seu ambiente (Python version, OS, etc.)

### Sugerir Melhorias

Sugestões são sempre bem-vindas! Abra uma Issue com:
- Descrição clara da melhoria
- Justificativa
- Exemplos se possível

### Contribuir com Código

1. **Fork o repositório**
   ```bash
   git clone https://github.com/YOUR_USERNAME/Algebra-Linear.git
   cd Algebra-Linear
   ```

2. **Crie uma branch para sua feature**
   ```bash
   git checkout -b feature/sua-feature-descritiva
   ```

3. **Desenvolva sua contribuição**
   - Mantenha o código limpo e bem documentado
   - Adicione comentários explicativos
   - Teste suas mudanças

4. **Commit com mensagens claras**
   ```bash
   git commit -m "Adiciona: descrição clara da mudança"
   ```

5. **Push para seu fork**
   ```bash
   git push origin feature/sua-feature-descritiva
   ```

6. **Abra um Pull Request**
   - Descreva as mudanças com clareza
   - Referencie issues relacionadas
   - Explique a motivação das mudanças

## 📋 Padrões de Código

- Use **PEP 8** para código Python
- Mantenha notebooks **bem documentados** com Markdown
- Use variáveis com **nomes descritivos**
- Adicione **comentários** em código complexo
- Teste suas mudanças antes de enviar

## 📚 Adicionando Novos Notebooks

Ao adicionar um novo notebook:

1. Siga o padrão de nomenclatura: `NN_Topico_Descritivo.ipynb`
2. Inclua:
   - Introdução clara do tópico
   - Conceitos teóricos
   - Exemplos práticos com visualizações
   - Exercícios propostos
3. Use visualizações para melhor compreensão
4. Adicione referências bibliográficas se necessário

## 🎨 Estilo e Estrutura

### Notebooks
- Primeira célula: Título e descrição
- Importar bibliotecas no início
- Usar Markdown para explicações
- Incluir visualizações
- Finalizar com resumo e exercícios

### Commits
```
Adiciona: descrição (para novas features)
Corrige: descrição (para bug fixes)
Melhora: descrição (para melhorias)
Documenta: descrição (para documentação)
Refatora: descrição (para refatoração)
```

## ✅ Checklist antes de enviar PR

- [ ] Código segue o padrão PEP 8
- [ ] Notebook está bem documentado
- [ ] Todas as células foram testadas
- [ ] Adicionei visualizações quando apropriado
- [ ] Mensagens de commit são claras
- [ ] Atualizei o README se necessário
- [ ] Não há conflitos com a branch main

## 📞 Dúvidas?

Se tiver dúvidas, abra uma **Issue** ou entre em contato!

Obrigado por contribuir! 🌟
